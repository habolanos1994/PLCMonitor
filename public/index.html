About the Project

This project consists of an API server and an MQTT client which both interact with PLCs (Programmable Logic Controllers) to collect data. The data is then published via MQTT and stored in a MongoDB database for further processing or analysis. The project is implemented with Node.js and uses the MQTT and worker_threads modules extensively.

the name of these service is DataCollectorwebsite.service and DataCollector.service

Key Features

  • Launches a worker thread for each PLC configuration read from a JSON file.
  • Each worker thread is set up with the PLC's name, IP address, and tag configurations, along with some other details such as emission frequency and plant location.
  • If any worker encounters an error or exits, it is logged into an error log.
  • Data from PLC is read and published via MQTT, and saved to a MongoDB database every half-hour.
  • If the PLC encounters an error or loses its connection, it tries to reconnect after a delay and logs the incident.
  • Unique serial numbers from each MQTT topic are counted and written to a CSV file.
  • Every day at 3:00 AM and 3:00 PM, it clears all the collected data and starts counting from zero again.
